# ربات ایجنت تلگرام + NVIDIA NIM + چرخش کلید + اجرای کد

## چیکار می‌کنه؟
- توی چت خصوصی تلگرام باهاش صحبت می‌کنی.
- برای هر کاربر، فقط **یک‌بار** دو تاپیک می‌سازه و برای همه‌ی پیام‌های بعدی همون دو تاپیک استفاده می‌شه (تاپیک جدید ساخته نمی‌شه):
  - 🧠 فکرها → مراحل داخلی، reasoning و خروجی ابزارها
  - 💬 پاسخ نهایی → جواب نهایی مدل (به‌صورت زنده/استریم ادیت می‌شه)
- از مدل‌های NVIDIA NIM استفاده می‌کنه؛ مدل پیش‌فرض و کل لیست قابل‌انتخاب توی `config.py` (`MODELS`) هست.
- اگه یک API key به محدودیت (rate limit) بخوره، خودکار میره سراغ کلید بعدی.
- پیام‌های چند کاربر مختلف به‌صورت **موازی** پردازش می‌شن (پیام‌های یک کاربر به ترتیب پردازش می‌شن).

## دستورات
| دستور | کاربرد |
|---|---|
| `/start` | ساخت دو تاپیک اختصاصی و راهنمای اولیه |
| `/stats` | تعداد پیام‌ها، کاربران فعال، کلیدهای فعال، مدل فعلی |
| `/models` | لیست کامل مدل‌های قابل‌انتخاب |
| `/model <name>` | تغییر مدل، مثلا `/model llama-3.1` |
| `/reset` | پاک کردن تاریخچه‌ی گفتگوی شما |
| `/export` | خروجی JSON از کل تاریخچه‌ی گفتگو |

## قابلیت‌ها
- **فایل**: هر فایلی که بفرستی دانلود و بررسی می‌شه (فایل‌های متنی مستقیم به مدل داده می‌شن، فایل‌های باینری از طریق ابزار `run_shell_command`/`run_python` قابل بررسی‌ان).
- **تصویر**: عکس‌ها به‌صورت base64 به مدل پاس داده می‌شن (نیازمند مدلی با پشتیبانی vision روی NIM).
- **اجرای کد**: `run_python` برای اجرای کد پایتون و `run_shell_command` برای اجرای هر زبان/دستور دیگه.
- **استریم پاسخ**: پیام پاسخ به‌جای ارسال یک‌جا، به‌صورت تدریجی ادیت می‌شه.
- **مانیتورینگ**: endpoint ساده‌ی `GET /health` روی پورت `PORT` (برای Railway healthcheck).
- **پارس‌مود**: پیام‌ها با `Markdown` فرستاده می‌شن (کد، بولد و ... درست رندر می‌شه)؛ اگه مارک‌داون مدل نامعتبر باشه، خودکار به متن ساده fallback می‌کنه.

## ۱. ساخت ربات تلگرام
1. با [@BotFather](https://t.me/BotFather) چت کن، `/newbot` بزن و توکن رو بگیر.
2. آیدی عددی خودت رو از [@userinfobot](https://t.me/userinfobot) بگیر (برای `ALLOWED_USER_IDS`).

> ⚠️ **حتما `ALLOWED_USER_IDS` رو پر کن.** چون این ربات دسترسی اجرای شل/پایتون داره، اگه بدون محدودیت بذاریش، هر کسی که آیدی ربات رو پیدا کنه می‌تونه روی سرورت کد اجرا کنه.

## ۲. گرفتن چند تا NVIDIA API Key
از [build.nvidia.com](https://build.nvidia.com) چند تا کلید (`nvapi-...`) بگیر — مثلاً با چند حساب مختلف — و همه رو با کاما توی `NVIDIA_API_KEYS` بذار.

## ۳. متغیرهای محیطی مهم (env vars)
```
TELEGRAM_BOT_TOKEN=...
ALLOWED_USER_IDS=123456789
NVIDIA_API_KEYS=nvapi-xxx,nvapi-yyy
DEFAULT_MODEL_KEY=glm-5.2
SHELL_ENABLED=true
ALLOW_DANGEROUS_COMMANDS=false
STREAM_ENABLED=true
MAX_CONCURRENT_UPDATES=8
DB_PATH=/data/bot.db
PORT=8080   # Railway خودش ست می‌کنه
```

## ۴. دیپلوی روی Railway
1. این پوشه رو به یه ریپازیتوری گیت‌هاب پوش کن.
2. توی Railway → New Project → Deploy from GitHub repo → همین ریپو رو انتخاب کن. Railway خودش `Dockerfile` رو تشخیص می‌ده.
3. برو تب **Variables** و همه‌ی متغیرهای بالا رو با مقدار واقعی ست کن.
4. (پیشنهادی) یه **Volume** بساز و به مسیر `/data` وصلش کن تا دیتابیس SQLite (تاریخچه و تاپیک‌ها) بین ری‌استارت‌ها از بین نره.
5. Deploy بزن. `railway.json` از `GET /health` برای healthcheck استفاده می‌کنه.

## ۵. تست
توی تلگرام به ربات پیام بده. باید فقط یک‌بار دو تاپیک بسازه و شروع کنه به جواب دادن؛ برای پیام‌های بعدی تاپیک جدیدی ساخته نمی‌شه.

## نکته درباره‌ی جستجوی وب
ابزار جستجوی سفارشی (DuckDuckGo scraping) از ربات حذف شده. اگه مدلی انتخاب کنی که خودش
قابلیت جستجوی وب داخلی (built-in / hosted search) داره، از همون استفاده می‌کنه؛ روی
endpoint استاندارد `/v1/chat/completions` مدل‌های فعلی NIM چنین قابلیتی به‌صورت پیش‌فرض
تضمین‌شده نیست، پس اگه به مدل خاصی با این قابلیت نیاز داری بگو تا `Config.MODELS` رو با
همون model id آپدیت کنیم.

## نکات امنیتی مهم
- `ALLOW_DANGEROUS_COMMANDS` رو روی `false` نگه دار مگر اینکه واقعاً بدونی چیکار می‌کنی.
- ابزارهای `run_shell_command` و `run_python` اجرای کد دلخواه روی همون کانتینر رو ممکن می‌کنن؛ `ALLOWED_USER_IDS` همیشه باید پر باشه.
- کانتینر Railway از سیستم اصلی تو ایزوله‌ست، ولی بازم اگه دیتابیس یا سرویس حساسی روی همون کانتینر داری، محتاط باش.
- توکن‌ها و کلیدها رو هیچ‌وقت داخل کد commit نکن؛ فقط از Variables استفاده کن. اگه یه توکن جایی (چت، کد، لاگ) لو رفت، همون لحظه از BotFather/NVIDIA ریوکش کن و یه توکن جدید بگیر.

## توسعه‌های بعدی که می‌شه اضافه کرد
- محدود کردن مسیر اجرای شل/پایتون به یه sandbox واقعی (مثلا کانتینر جدا یا gVisor).
- ذخیره‌ی فایل‌های آپلودشده روی S3/Volume به‌جای فقط پردازش لحظه‌ای.
- Rate limiting در سطح هر کاربر برای جلوگیری از سوءاستفاده از ابزار اجرای کد.
