# ربات ایجنت تلگرام + NVIDIA NIM (GLM-5.2) + چرخش کلید + اجرای شل

## چیکار می‌کنه؟
- توی چت خصوصی تلگرام باهاش صحبت می‌کنی.
- برای هر کاربر، دو **تاپیک** می‌سازه:
  - 🧠 فکرها → مراحل داخلی، reasoning و اجرای دستورات شل
  - 💬 پاسخ نهایی → جواب نهایی مدل
- از مدل `z-ai/glm-5.2` روی NVIDIA NIM استفاده می‌کنه که هم reasoning داره هم tool calling.
- اگه یک API key به محدودیت (rate limit) بخوره، خودکار میره سراغ کلید بعدی.
- ایجنت می‌تونه دستور شل (`run_shell_command`) روی همون کانتینر اجرا کنه.

## ۱. ساخت ربات تلگرام
1. با [@BotFather](https://t.me/BotFather) چت کن، `/newbot` بزن و توکن رو بگیر.
2. آیدی عددی خودت رو از [@userinfobot](https://t.me/userinfobot) بگیر (برای `ALLOWED_USER_IDS`).

> ⚠️ **حتما `ALLOWED_USER_IDS` رو پر کن.** چون این ربات دسترسی اجرای شل داره، اگه بدون محدودیت بذاریش، هر کسی که آیدی ربات رو پیدا کنه می‌تونه روی سرورت دستور اجرا کنه.

## ۲. گرفتن چند تا NVIDIA API Key
از [build.nvidia.com](https://build.nvidia.com) چند تا کلید (`nvapi-...`) بگیر — مثلاً با چند حساب مختلف — و همه رو با کاما توی `NVIDIA_API_KEYS` بذار.

## ۳. دیپلوی روی Railway
1. این پوشه رو به یه ریپازیتوری گیت‌هاب پوش کن.
2. توی Railway → New Project → Deploy from GitHub repo → همین ریپو رو انتخاب کن. Railway خودش `Dockerfile` رو تشخیص می‌ده.
3. برو تب **Variables** و همه‌ی متغیرهای داخل `.env.example` رو با مقدار واقعی ست کن.
4. (پیشنهادی) یه **Volume** بساز و به مسیر `/data` وصلش کن تا دیتابیس SQLite (تاریخچه و تاپیک‌ها) بین ری‌استارت‌ها از بین نره.
5. Deploy بزن. توی تب **Console** می‌تونی لاگ‌ها رو ببینی یا دستی دستور اجرا کنی.

## ۴. تست
توی تلگرام به ربات پیام بده. باید خودش دو تاپیک بسازه و شروع کنه به جواب دادن.

## نکات امنیتی مهم
- `ALLOW_DANGEROUS_COMMANDS` رو روی `false` نگه دار مگر اینکه واقعاً بدونی چیکار می‌کنی.
- کانتینر Railway از سیستم اصلی تو ایزوله‌ست، ولی بازم اگه دیتابیس یا سرویس حساسی روی همون کانتینر داری، محتاط باش.
- توکن‌ها و کلیدها رو هیچ‌وقت داخل کد commit نکن؛ فقط از Variables استفاده کن.

## توسعه‌های بعدی که می‌شه اضافه کرد
- استریم کردن پاسخ با `sendMessageDraft` (تازه توی Bot API اضافه شده) به جای فرستادن یه پیام کامل.
- محدود کردن مسیر اجرای شل به یه پوشه‌ی sandbox خاص.
- اضافه کردن دستورات تلگرام مثل `/reset` برای پاک کردن تاریخچه یه کاربر.
